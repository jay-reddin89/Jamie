## 2026-02-25 - [Keyboard Accessibility and Semantic HTML]
**Learning:** Replacing interactive `div` elements with semantic `<button>` elements and adding `:focus-visible` styles significantly improves the experience for keyboard and screen reader users without requiring large redesigns.
**Action:** Always prefer `<button>` for interactive elements and use a `.btn-reset` utility class to maintain visual flexibility while ensuring keyboard accessibility.
