## 2025-05-14 - Enhancing Form Accessibility and Keyboard Navigation

**Learning:** Associating `<label>` elements with their respective `<input>` elements using the `for` attribute is a fundamental yet often overlooked accessibility requirement. It not only assists screen readers but also significantly improves the user experience by expanding the interactive click/tap area to include the label itself. Additionally, implementing `:focus-visible` provides a clear, high-contrast visual indicator for keyboard users, ensuring the application remains navigable without a mouse.

**Action:** Always ensure every form input has a corresponding `<label>` with a matching `for` attribute and that all interactive elements have well-defined `:focus-visible` styles to support inclusive design.
