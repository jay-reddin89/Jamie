## 2025-05-15 - Hyper-focused Accessibility and UX Improvements
**Learning:** Combining multiple accessibility and UX improvements (e.g., form labels, focus-visible styles, and ARIA attributes) into a single PR can be perceived as scope creep and violate repository hygiene constraints, especially when a 50-line limit is in place.
**Action:** Prioritize exactly one micro-UX win per PR (e.g., focusing solely on form accessibility) and strictly maintain repository hygiene by ensuring build artifacts and logs are excluded from the git stage.
