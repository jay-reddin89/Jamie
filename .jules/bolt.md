## 2025-05-14 - Optimizing High-Frequency Live Updates
**Learning:** High-frequency `setInterval` loops (1000ms) that perform $O(1)$ operations like `new Date()` and `document.getElementById` can accumulate significant overhead and main-thread noise. Specifically, `Intl.NumberFormat` and `toLocaleDateString` are relatively expensive when re-initialized or called on every tick. Defining helper functions *inside* these loops also causes unnecessary function re-allocation.
**Action:** Always hoist constant derivations (like `bornDay`), persistent formatters (`Intl.NumberFormat`), and DOM element caches outside of update loops. Implement dirty checking via strict inequality (`!==`) on `textContent` to prevent redundant DOM mutations. Define helper functions (e.g., `updateStat`) outside the loop scope to minimize allocation overhead.
